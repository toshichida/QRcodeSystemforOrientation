# QRコード受付システム アーキテクチャ設計書

## 1. システム概要

本システムは、留学生オリエンテーションの受付を効率化するためのQRコードベースの受付システムである。Google Apps Script（GAS）をバックエンド、GitHub Pages でホスティングする静的Webアプリをフロントエンドとして構成する。

## 3. コンポーネント一覧

### 3.1 フロントエンド（QR読み取りアプリ）

| 項目 | 内容 |
|------|------|
| **技術** | HTML5, JavaScript, QR読み取りライブラリ（jsQR / html5-qrcode 等） |
| **ホスティング** | GitHub Pages |
| **デプロイ** | `gh` コマンドでリポジトリ作成・プッシュ・GitHub Pages 有効化 |
| **責務** | カメラによるQR読み取り、参加者情報表示、受付登録API呼び出し、モード切り替え、音声フィードバック |

### 3.2 バックエンド（GAS Web アプリ）

| 項目 | 内容 |
|------|------|
| **技術** | Google Apps Script (JavaScript) |
| **デプロイ** | Google Apps Script エディタ上で手動デプロイ |
| **コード管理** | Clasp で紐付け済みプロジェクトにプッシュ |
| **責務** | スプレッドシート初期設定、REST API提供、スプレッドシート読み書き、メール送信、QRコード生成 |

### 3.3 データストア

| 項目 | 内容 |
|------|------|
| **Google スプレッドシート** | 参加者マスターデータ、メール設定の永続化 |
| **Gmail** | メール下書きの一時保存、送信 |

### 3.4 外部サービス

| 項目 | 内容 |
|------|------|
| **GoQR API** | QRコード画像の生成（メール本文挿入用） |

## 4. データフロー

### 4.1 事前準備フロー（メール送付）

```
管理者
  │
  ├─ 0. （初回のみ）GAS メニューから「スプレッドシート初期設定」を実行し、シート・ヘッダーを自動作成
  │
  ├─ 1. スプレッドシートに参加者データ入力
  │
  ├─ 2. メール設定ダイアログで設定
  │
  ├─ 3. 「下書き一括作成」実行
  │      └─ GAS → Gmail 下書き作成
  │      └─ GAS → 参加者データ「下書き作成日時」更新
  │
  ├─ 4. 「一斉送信」実行
  │      └─ GAS → Gmail 送信
  │      └─ GAS → 参加者データ「送信日時」「送信ステータス」更新
  │
  └─ 5. 参加者にQRコード付きメールが届く
```

### 4.2 受付フロー（QRスキャン）

```
受付担当者 / 学生
  │
  ├─ 1. QR読み取りアプリを起動（午前/午後モード選択）
  │
  ├─ 2. 担当者名を選択（午前/午後共通）
  │
  ├─ 3. QRコードをスキャン
  │      └─ フロントエンド: カメラでQR読み取り → ID 抽出
  │
  ├─ 4. GET /participant?id={id} で参加者情報取得
  │      └─ GAS → スプレッドシート検索 → JSON 返却
  │
  ├─ 5a. ID が存在する場合
  │      ├─ 参加者情報を画面表示
  │      ├─ POST /reception で受付登録
  │      │      └─ GAS → スプレッドシート「受付ステータス」「受付日時」「受付担当者」更新
  │      ├─ スキャン成功音
  │      └─ 既に受付済みなら「受付済み」表示
  │
  └─ 5b. ID が存在しない場合
         ├─ エラー表示
         └─ エラー音
```

## 5. セキュリティ・アクセス制御

### 5.1 GAS Web アプリ

| 項目 | 方針 |
|------|------|
| **デプロイ時のアクセス** | 「全員」でデプロイ（URLを課内スタッフにのみ共有） |
| **CORS** | フロントエンド（GitHub Pages）からのアクセスを許可するため、`ContentService` で適切なヘッダを設定 |
| **認証** | 現状はURLの非公開性に依存。必要に応じて「組織内」やパスワード追加を検討 |

### 5.2 フロントエンド

| 項目 | 方針 |
|------|------|
| **HTTPS** | GitHub Pages は HTTPS 必須（カメラAPI利用のため） |
| **API URL** | フロントエンドにGAS WebアプリURLを埋め込む。環境変数や設定ファイルで管理を検討 |

## 6. デプロイ・運用

### 6.1 デプロイ方式

| コンポーネント | デプロイ方法 |
|----------------|--------------|
| **GAS** | Clasp でコードをプッシュ → GAS エディタ上で手動デプロイ |
| **フロントエンド** | `gh` でリポジトリ作成 → プッシュ → GitHub Pages 有効化 |

### 6.2 環境

| 環境 | 用途 |
|------|------|
| **本番** | オリエンテーション当日の運用 |
| **開発/テスト** | 開発時は別スプレッドシート・別GASプロジェクトを推奨 |

## 7. 技術的制約・前提

| 項目 | 内容 |
|------|------|
| **ブラウザ** | カメラAPI対応ブラウザ（Chrome, Safari, Edge 等） |
| **ネットワーク** | オンライン必須（API呼び出しのため） |
| **同時接続** | 窓口2口 × 午前・午後を想定。GASの同時実行制限に注意 |
| **GAS 実行時間** | 1リクエスト6分制限。本システムの処理は軽量のため問題なし |
