# GoQR API・HTML5 QRコードライブラリ 技術選定チェック結果

本ドキュメントは、要件定義書で採用予定の **GoQR API** および **HTML5 QRコード読み取りライブラリ**（jsQR / html5-qrcode）について、最新情報に基づく実装可否を確認した結果です。

**調査日**: 2025年3月

---

## 1. GoQR API（QRコード画像生成）

### 1.1 概要

| 項目 | 内容 |
|------|------|
| **提供元** | goQR.me（Foundata GmbH） |
| **公式ドキュメント** | https://goqr.me/api/doc/ |
| **create-qr-code 詳細** | https://goqr.me/api/doc/create-qr-code/ |

### 1.2 最新の利用可否

✅ **利用可能** — 公式ドキュメントが公開されており、サービスは稼働中です。

### 1.3 API 仕様（実装時に使用する値）

| 項目 | 値 |
|------|-----|
| **ベースURL** | `https://api.qrserver.com/v1/create-qr-code/` |
| **プロトコル** | **HTTPS を推奨**（HTTP も可。GAS からの呼び出しは HTTPS 推奨） |
| **メソッド** | GET（または POST） |
| **必須パラメータ** | `data` … エンコードするテキスト（URL エンコード必須） |
| **任意パラメータ** | `size`（例: 200x200）、`format`（png/gif/jpeg/svg/eps）、`ecc`（L/M/Q/H）、`charset-source`（UTF-8）、`charset-target`（UTF-8） |

#### 使用例（GAS でのメール本文挿入用）

```
https://api.qrserver.com/v1/create-qr-code/?data=ORI-001&size=200x200&format=png
```

HTML メール内の `<img>` タグで直接参照可能:

```html
<img src="https://api.qrserver.com/v1/create-qr-code/?data=ORI-001&size=200x200&format=png" alt="QR Code" />
```

### 1.4 制限・注意事項

| 項目 | 内容 |
|------|------|
| **リクエスト制限** | 公式の上限なし。1日10,000リクエスト超の場合は事前連絡推奨 |
| **本システムの想定** | 150〜200名 × 1通 = 約200リクエスト/回。問題なし |
| **データ長** | 最大約900文字（参加者ID程度なら十分） |
| **認証** | API キー不要 |

### 1.5 実装時の推奨

- **HTTPS** を必ず使用する（`https://api.qrserver.com/...`）
- `data` パラメータは `encodeURIComponent()` で URL エンコードする
- 日本語などの特殊文字は `charset-source=UTF-8` を指定（参加者ID が英数字のみなら不要）

## 2. フロントエンド QR 読み取りライブラリ

### 2.1 比較: jsQR vs html5-qrcode

| 項目 | jsQR | html5-qrcode |
|------|------|--------------|
| **バージョン** | 1.4.0 | 2.3.8 |
| **週間ダウンロード** | 約110万 | 約72万 |
| **メンテナンス** | 2023年12月更新 | 継続的に更新 |
| **カメラ対応** | 自前で実装（getUserMedia + canvas） | 標準 API で完結 |
| **UI** | カスタムUI 必須 | 簡易UI / カスタムUI 両対応 |
| **ドキュメント** | GitHub README | scanapp.org で公式ドキュメント |
| **CDN** | あり | `https://unpkg.com/html5-qrcode` |

### 2.2 推奨: html5-qrcode

本プロジェクトでは **html5-qrcode** を推奨します。

**理由**:
- カメラ取得・ストリーム表示が API 内で完結している
- `Html5Qrcode` でカスタムUI を組みやすい（午前/午後モード切り替えに適する）
- `facingMode: "environment"` でスマートフォンの背面カメラを指定可能
- 公式ドキュメントが充実（https://scanapp.org/html5-qrcode-docs/docs/intro）

### 2.3 html5-qrcode 導入方法

#### CDN（GitHub Pages 向け）

```html
<script src="https://unpkg.com/html5-qrcode" type="text/javascript"></script>
```

#### カスタムUI での実装例（Pro Mode）

```javascript
const html5QrCode = new Html5Qrcode("reader");
html5QrCode.start(
  { facingMode: "environment" },  // スマホ背面カメラ
  { fps: 10, qrbox: { width: 250, height: 250 } },
  (decodedText, decodedResult) => {
    // decodedText が参加者ID
    onScanSuccess(decodedText);
  },
  (errorMessage) => { /* エラー時は無視して継続 */ }
);
```

### 2.4 実装時の注意

| 項目 | 内容 |
|------|------|
| **HTTPS** | カメラ API は HTTPS または localhost 必須（GitHub Pages は HTTPS 対応） |
| **ユーザー許可** | ページ読み込み直後ではなく、ユーザー操作後にカメラ許可を求める |
| **停止処理** | `html5QrCode.stop()` でカメラを停止する（モード切り替え時など） |

## 3. 結論・実装への反映

| 技術 | 判定 | 実装時のポイント |
|------|------|------------------|
| **GoQR API** | ✅ 採用可 | HTTPS 使用、`encodeURIComponent` で data をエンコード |
| **html5-qrcode** | ✅ 推奨 | CDN で導入、Pro Mode でカスタムUI を実装 |
| **jsQR** | △ 代替可 | カメラ・canvas の自前実装が必要 |
