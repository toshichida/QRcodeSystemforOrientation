# QRコード受付システム リポジトリ構造の定義書

## 1. 概要

本プロジェクトは **単一リポジトリ（モノレポ）** で、以下を包含する。

- **GAS（Google Apps Script）**: Clasp で紐付け済みプロジェクトにプッシュ
- **フロントエンド（QR読み取りアプリ）**: GitHub Pages でデプロイ
- **ドキュメント**: 設計書・要件定義書

---

## 3. 各ディレクトリの役割

### 3.1 ルート直下

| ファイル/ディレクトリ | 説明 |
|----------------------|------|
| `.clasp.json` | Clasp の設定。`scriptId` で GAS プロジェクトと紐付け。`rootDir` で `gas/` を指定 |
| `.claspignore` | Clasp プッシュ時に除外するファイル（`docs/`, `README.md`, `index.html`, `css/`, `js/`, `assets/` 等） |
| `appsscript.json` | ルートに配置する場合は Clasp のデフォルト用。`rootDir: gas` の場合は `gas/appsscript.json` を参照 |
| `README.md` | プロジェクト概要、セットアップ手順 |

### 3.2 gas/

| 項目 | 説明 |
|------|------|
| **役割** | GAS のソースコード。スプレッドシート初期設定、Web API、メール送信、スプレッドシート操作 |
| **デプロイ** | Clasp でプッシュ → GAS エディタ上で手動デプロイ |
| **管理** | `.clasp.json` の `rootDir` を `gas` に設定 |

### 3.3 frontend/

| 項目 | 説明 |
|------|------|
| **役割** | QRコード読み取りアプリ（HTML5 + JavaScript） |
| **デプロイ** | GitHub にプッシュ → GitHub Pages で公開 |
| **設定** | `config.js` に GAS Web アプリ URL を記載 |

### 3.4 docs/

| 項目 | 説明 |
|------|------|
| **役割** | 要件定義、アーキテクチャ、API、リポジトリ構造の設計書 |
| **デプロイ** | リポジトリに含めるのみ（公開不要の場合は .gitignore で除外可） |

## 4. Clasp 設定

### 4.1 .clasp.json の例

```json
{
  "scriptId": "YOUR_SCRIPT_ID",
  "rootDir": "gas"
}
```

- `scriptId`: GAS プロジェクトの ID（GAS エディタの「プロジェクトの設定」で確認）
- `rootDir`: GAS ソースが格納されているディレクトリ

### 4.2 .claspignore の例

```
frontend/
docs/
README.md
.git/
node_modules/
```

### 4.3 Clasp コマンド

| コマンド | 説明 |
|----------|------|
| `clasp push` | `gas/` の内容を GAS プロジェクトにプッシュ |
| `clasp pull` | GAS プロジェクトの内容を `gas/` に取得 |
| `clasp open` | GAS エディタをブラウザで開く |

## 5. GitHub 運用（gh コマンド）

### 5.1 前提

- GitHub CLI (`gh`) がインストール済み
- `gh auth login` で認証済み

### 5.2 リポジトリ作成

```bash
# リポジトリを新規作成（プライベート推奨）
gh repo create QRcodeSystemforOrientation --private --source=. --remote=origin

# または既存のローカルプロジェクトを GitHub に紐付ける場合
gh repo create QRcodeSystemforOrientation --private --source=. --push
```

### 5.3 プッシュ

```bash
git add .
git commit -m "Initial commit"
git push -u origin main
```

### 5.4 GitHub Pages の有効化

```bash
# Settings → Pages で設定するか、gh で設定
gh api repos/:owner/:repo/pages -X PUT -f source='{"branch":"main","path":"/frontend"}'
```

※ `path` が `/frontend` の場合、GitHub Pages は `frontend/` をルートとして公開する。  
※ GitHub Pages の「Source」で「Deploy from a branch」を選択し、Branch を `main`、Folder を `/frontend` に設定する。

**注意**: GitHub Pages はルートまたは `/docs` のみをソースとして選択できる場合がある。`/frontend` をルートとして使う場合は、以下のいずれかとする。

- **方式A**: リポジトリルートに `frontend/` の内容を配置し、ルートを GitHub Pages のソースにする
- **方式B**: `frontend/` を別リポジトリとして分離し、そのリポジトリで GitHub Pages を有効化
- **方式C**: `main` ブランチのルートをサイトのルートとし、`index.html` をルートに配置（`frontend/` の内容をルートに移動）

### 5.5 推奨構造（GitHub Pages 対応）

GitHub Pages は「ルート」「/docs」「gh-pages ブランチ」から選択するため、次のいずれかを推奨する。

**推奨: frontend をルートに配置**

```
QRcodeSystemforOrientation/
├── index.html              # QRアプリのエントリ（frontend の index.html）
├── css/
├── js/
├── assets/
├── gas/                    # GAS ソース
├── docs/                   # 設計書
├── .clasp.json
└── ...
```

この場合、`frontend/` を廃止し、QRアプリのファイルをルート直下に配置する。

**代替: /docs を GitHub Pages に使用しない**

`docs/` は設計書用として保持し、GitHub Pages のソースは `frontend/` とする。  
GitHub の「Pages」設定で `frontend` フォルダを指定できない場合は、**サブモジュール**や**別リポジトリ**で QR アプリを管理する。

**実務的な推奨**: ルートに QR アプリ（index.html, css/, js/, assets/）を配置し、`gas/` と `docs/` をサブディレクトリとして保持する。

## 7. デプロイ手順まとめ

### 7.1 GAS のデプロイ

```bash
# 1. gas/ のコードを GAS にプッシュ
clasp push

# 2. ブラウザで GAS エディタを開く
clasp open

# 3. GAS エディタ上で「デプロイ」→「新しいデプロイ」→「ウェブアプリ」で手動デプロイ
# 4. デプロイ URL を取得し、frontend の config.js に設定
```

### 7.2 フロントエンド（GitHub Pages）のデプロイ

```bash
# 1. リポジトリ作成（初回のみ）
gh repo create QRcodeSystemforOrientation --private --source=. --push

# 2. 変更をプッシュ
git add .
git commit -m "Update QR scanner"
git push

# 3. GitHub の Settings → Pages で「Deploy from a branch」を選択
#    Branch: main, Folder: / (root)
# 4. 数分後に https://{owner}.github.io/QRcodeSystemforOrientation/ で公開
```

### 7.3 初回セットアップ一覧

| 順序 | 作業 | コマンド/操作 |
|------|------|---------------|
| 1 | GAS プロジェクト作成 | Google スプレッドシートから「拡張機能」→「Apps Script」 |
| 2 | Clasp で紐付け | `clasp clone {scriptId}` または `.clasp.json` 作成 |
| 3 | GAS コードをプッシュ | `clasp push` |
| 4 | スプレッドシート初期設定 | GAS エディタでスプレッドシートを開き、メニューから「スプレッドシート初期設定」を実行 |
| 5 | GAS Web アプリをデプロイ | GAS エディタで手動デプロイ |
| 6 | GitHub リポジトリ作成 | `gh repo create ...` |
| 7 | コードをプッシュ | `git push` |
| 8 | GitHub Pages 有効化 | Settings → Pages |

## 8. 環境変数・設定

| 設定項目 | 格納場所 | 説明 |
|----------|----------|------|
| GAS Web アプリ URL | `js/config.js` | フロントエンドが API を呼び出す際のベース URL |
| Clasp scriptId | `.clasp.json` | GAS プロジェクト ID（リポジトリに含める場合は注意） |

※ `.clasp.json` の `scriptId` は機密性は低いが、必要に応じて `.gitignore` に追加し、テンプレート（`.clasp.json.example`）をリポジトリに含める。
